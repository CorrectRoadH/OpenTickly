# Cloud / Self-Hosted Parity Policy

## 目的

定义 `OpenToggl` 云 SaaS 与自托管版之间“功能面一致”到底是什么意思，避免它停留在口号层。

## 当前结论

- 云版与自托管版对外公开的产品定义、API 契约、对象模型和 Web 功能面应保持一致。
- 不允许把自托管版定义为裁剪版社区版。
- 允许云版拥有额外运维设施，但不应形成对外功能差异。

## 待补充的关键约束

- “功能一致”具体指哪些层：
  - API 形状
  - 行为语义
  - 商业能力
  - 运维能力
  - 第三方支付/通知依赖
- 哪些能力允许通过不同执行模型实现，但仍视为对外兼容。
- billing / quota / invoice / payment 在自托管版中的执行模型。
- reports / webhooks / import 在两种部署模型下的一致性边界。
- 是否允许自托管版采用 BYO 外部依赖。

## 与 PRD 的关系

该文档用于支撑 `docs/prd.md` 中“云 SaaS 与自托管功能面一致”的承诺。
