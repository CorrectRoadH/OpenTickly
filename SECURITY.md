# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately through GitHub's [Private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).

**Do not open a public issue.**

You can expect:

- An initial acknowledgement within **8 hours**
- A preliminary assessment within **24 hours**
- A fix or mitigation plan communicated as soon as it's ready

Confirmed vulnerabilities will be publicly credited to the reporter after a fix is released, unless you prefer to remain anonymous.
