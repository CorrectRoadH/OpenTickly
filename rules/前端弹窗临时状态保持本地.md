# 弹窗临时表单状态保持本地

只在弹窗或浮层打开期间有意义的字段状态，应保留在对应弹窗或浮层内部。除非父组件确实需要响应编辑中的值，否则不要把 `[name, setName]`、`[email, setEmail]`、选中颜色、开关或步骤状态提升到页面或外壳组件。

```grit
language js
`<$Dialog $field={$value} $handler={$setValue} />` where {
  $filename <: r".*apps/website/src/pages/.*Page\.tsx",
  $Dialog <: r".*Dialog$",
  $handler <: r"^on[A-Z].*Change$",
  $setValue <: r"^set[A-Z].*"
}
```

## 反例

```tsx
function Page() {
  const [email, setEmail] = useState("");
  return <InviteDialog email={email} onEmailChange={setEmail} />;
}
```

## 正例

```tsx
function Page() {
  return <InviteDialog onSubmit={(values) => mutate(values)} />;
}
```
