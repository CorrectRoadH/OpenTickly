OpenToggl's AI integration is built around [`CorrectRoadH/toggl-cli`](https://github.com/CorrectRoadH/toggl-cli) — a CLI maintained specifically for AI agents and automation. Two ways to use it:

- `toggl` CLI — operate directly from the command line
- `SKILL.md` skill pack — install into an agent environment

## Install

For agent environments:

```bash
npx skills add CorrectRoadH/toggl-cli
```

Manual CLI install:

```bash
npm install -g @correctroadh/toggl-cli
toggl --help
```

## What the CLI covers

Once installed, agents can run common operations as commands instead of crafting raw HTTP requests:

- Time entries: `start`, `stop`, `continue`, `running`, `show`, `edit`, `delete`
- Workspaces: `list`, `create`, `rename`, `delete`
- User info: `me`, `preferences`
- Organizations: `organization list`, `organization show`

## Connecting to OpenToggl

The CLI supports both official Toggl Track and OpenToggl self-hosted. Choose during first-time auth:

- Toggl Track → connects to the official service
- OpenToggl → enter your instance URL, e.g. `https://your-instance.com/api/v9`

## Why a CLI instead of raw API calls

- Commands map directly to common tasks — agents don't need to understand the underlying API
- Built-in auth, no manual token management
- Local caching for reads, automatic cache invalidation on writes
- Skill pack can be installed and reused across agent environments

## Recommended setup

1. Deploy OpenToggl (or use the hosted version)
2. Install toggl-cli or the skill pack
3. Authenticate to your instance
4. Agents use CLI commands for day-to-day operations
5. For finer control, refer to the OpenAPI docs
