OpenTickly은 지속적으로 릴리스됩니다. 각 릴리스의 권위 있는 출처는 GitHub Releases이며, 여기에 diff·커밋 범위·업그레이드 노트가 모두 정리되어 있습니다.

- 최신 및 이전 릴리스: [github.com/CorrectRoadH/OpenTickly/releases](https://github.com/CorrectRoadH/OpenTickly/releases)
- 전체 커밋 히스토리: [github.com/CorrectRoadH/OpenTickly/commits/main](https://github.com/CorrectRoadH/OpenTickly/commits/main)
- Docker 이미지: [hub.docker.com/r/correctroad/opentickly](https://hub.docker.com/r/correctroad/opentickly)

## 릴리스 절차

OpenTickly은 GitHub에서 공개적으로 개발됩니다. 각 릴리스는 `main`의 태그된 커밋에 대응하며 자동으로 Docker Hub에 게시되고, 호스팅 데모 [track.opentoggl.com](https://track.opentoggl.com)에도 함께 배포됩니다. 셀프호스팅 사용자는 새 이미지를 pull 하고 `docker compose up -d`를 다시 실행하면 업그레이드됩니다.

## 업그레이드 유의사항

대부분의 릴리스는 드롭인 업그레이드가 가능합니다. Toggl Track API 표면은 안정적이고, Postgres 스키마는 시작 시 자동으로 앞으로 마이그레이션됩니다. 호환성을 깨는 변경이 있다면 해당 GitHub Release 안에 별도의 **"Upgrade"** 섹션과 마이그레이션 계획이 명시됩니다.

업그레이드에 실패하면 [github.com/CorrectRoadH/OpenTickly/issues](https://github.com/CorrectRoadH/OpenTickly/issues)에 릴리스 태그와 실패 로그를 함께 올려 이슈를 열어 주세요.

## API 호환성 약속

OpenTickly은 Toggl Track v9, Reports v3, Webhooks v1의 공개 계약을 따라갑니다. 기존 Toggl 연동, CLI, MCP 도구는 릴리스가 바뀌어도 수정 없이 계속 동작해야 합니다. 상류 Toggl에서 엔드포인트가 추가되거나 조정되면, OpenTickly은 다음 릴리스 사이클 안에 따라가는 것을 목표로 합니다.
