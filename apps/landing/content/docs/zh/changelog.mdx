OpenToggl 持续迭代。每次发布的权威来源是 GitHub Releases，里面有完整 diff、commit 范围和升级说明。

- 历次发布：[github.com/CorrectRoadH/opentoggl/releases](https://github.com/CorrectRoadH/opentoggl/releases)
- 完整 commit 历史：[github.com/CorrectRoadH/opentoggl/commits/main](https://github.com/CorrectRoadH/opentoggl/commits/main)
- Docker 镜像：[hub.docker.com/u/correctroadh](https://hub.docker.com/u/correctroadh)

## 发布流程

OpenToggl 在 GitHub 上公开开发。每次发布对应 `main` 上的一个 tag commit，会自动推送到 Docker Hub，同时同步到托管演示站 [track.opentoggl.com](https://track.opentoggl.com)。自托管用户拉新镜像、重新 `docker compose up -d` 即可升级。

## 升级注意事项

大多数版本是平滑升级：Toggl Track API 表面稳定，Postgres schema 在启动时自动向前迁移。如果某次发布包含不兼容改动，对应的 GitHub Release 里会有明确的 **"Upgrade"** 段落和迁移步骤。

如果升级失败，在 [github.com/CorrectRoadH/opentoggl/issues](https://github.com/CorrectRoadH/opentoggl/issues) 提 issue，附上版本号和失败日志。

## API 兼容性承诺

OpenToggl 对齐 Toggl Track v9、Reports v3、Webhooks v1 的公开契约。已有的 Toggl 集成、CLI、MCP 工具在后续版本里应该继续能用。上游 Toggl 新增或调整接口时，OpenToggl 会争取在一个发布周期内跟进。
