下面这些是作者日常怎么用 OpenToggl 的

## 让 Agent 帮你开计时

我不会自己去 Web UI 点"开始"按钮。我会跟 OpenClaw 说一句"我现在要去做什么事情"，Agent 通过 [`toggl-cli`](https://github.com/CorrectRoadH/toggl-cli) 启动一个 time entry。 然后如果我不会主动结束，如果我切换了工作内容，我会说"我现在要去做另外一件事了"，toggl 启动一个新的 time entry，便会自动把之前那个结束了。

## 把周计划和原则塞给 Agent

我把自己写在 Obsidian 里的个人使命、本周目标、日常原则一起给 Agent 当上下文。所以当我跟它说"我要刷两个小时小红书"的时候，它会回我："你本周的计划是写完 xxx 功能，现在是不是要改做 yyy？"

同时我使用 4 象限管理方法，所以我一共有 6 个 Tag， 重要、不重要、紧急、不紧急 与 专心、不专心。前面四个 tag， AI 会在事情开始时为我打上，后面两个 tag 是在事情结束时，再多问我一句是否专心了。这样我就能在每天结束的时候，看到自己时间分布在不同象限的比例，以及专心和不专心的比例。

## 每晚 cron 让 AI 复盘

我挂了一个 cron 任务，每晚自动拉出当天的时间分布，让 AI 做一次复盘：

- 不同象限（重要 / 紧急）的时间比例合不合理？
- 今天比昨天、比本周均值有没有进步？
- 有没有出现本不该出现的时间黑洞？
- 今天的工作是否偏离了本周的计划？

这件事在官方 Toggl 上做起来比较麻烦。因为有 30 次/小时的 API 限制，可能一次复盘时 AI 想多看几次就能把配额打光了。
