下面这些是作者日常怎么用 OpenToggl 的，给你当参考。不是标准流程，自己改着用。

## 让 Agent 帮你开计时

我不会自己去 Web UI 点"开始"按钮。我直接跟 Claude / OpenClaw 说一句"我现在要写 xxx 代码"或者"我要看一会儿 b 站"，Agent 通过 [`toggl-cli`](https://github.com/CorrectRoadH/toggl-cli) 就把计时器开起来了。

好处是一句话同时做了两件事 —— 说出意图 + 开始计时，不用切窗口。写代码本来就在跟 AI 对话，顺手就把计时也做了，几乎没有额外成本。

安装见 [AI 集成](/zh/docs/ai-integration)。

## 把周计划和原则塞给 Agent

我把自己写在 Obsidian 里的个人使命、本周目标、日常原则一起给 Agent 当上下文。所以当我跟它说"我要刷两个小时小红书"的时候，它会回我："你本周的计划是写完 xxx 功能，现在是不是要改做 yyy？"

Agent 不是监工，也不会硬拦着你。它只是帮你意识到"你正在偏离自己设定的方向"。最终做不做，还是你自己决定。

## 每晚 cron 让 AI 复盘

我挂了一个 cron 任务，每晚自动拉出当天的时间分布，让 AI 做一次复盘：

- 不同象限（重要 / 紧急）的时间比例合不合理？
- 今天比昨天、比本周均值有没有进步？
- 有没有出现本不该出现的时间黑洞？

这件事在官方 Toggl 上是做不起来的 —— 30 次/小时的 API 限制，一次复盘就把配额打光了。OpenToggl 自己部署没这个限制，AI 爱跑多少轮就跑多少轮。
